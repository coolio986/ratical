## The two rules that save you the most pain

1. **Never edit a file whose header says `# WARNING. DO NOT EDIT THIS FILE.`** Those are
   generated or bundled. Put overrides in your top-level `printer.cfg` or in
   `configuration/macros/overrides.cfg`.
2. **Boards and printers are auto-discovered as data files, not registered in code.** To
   add one you drop a folder into `configuration/boards/` or `configuration/printers/`
   that matches the schema — see the guides above.
