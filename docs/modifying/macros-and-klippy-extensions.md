# Modifying Macros and klippy Extensions

Two related but different things live under `configuration/`:

- **Macros** — G-code macros in `configuration/macros/*.cfg`. Pure Klipper config. No
  rebuild, no registration; a klipper restart picks them up.
- **klippy extensions** — Python modules in `configuration/klippy/*.py` that add new
  behavior to Kalico itself (new commands, kinematics, probes). These must be **registered**
  and **symlinked** into the klipper checkout.

> **Prerequisite reading:** [ARCHITECTURE.md §3](../ARCHITECTURE.md#3-configuration--the-klipper-payload)
> and the [glossary](../glossary.md) entries for *klippy*, *kinematics*, *Beacon*.

## Part A — Macros

### Layout (`configuration/macros/`)

| File / dir | What it does |
|---|---|
| `calibration.cfg` | Probe/mesh/offset calibration macros. |
| `mesh.cfg` | Bed-mesh macros. |
| `parking.cfg` | Toolhead parking. |
| `priming.cfg` | Prime line / purge. |
| `heatsoaking.cfg` | Chamber/bed soak. |
| `commissioning.cfg` | First-run / `RATICAL_COMMISSION` wizard. |
| `load_filament.cfg` / `unload_filament.cfg` | Filament handling. |
| `led_control.cfg` | Status LEDs. |
| `chamber_filter.cfg` / `chamber_heater.cfg` | Chamber accessories. |
| `idex/` | IDEX-specific macros (tool change, copy/mirror). |
| `util.cfg` | Shared helper macros. |
| **`overrides.cfg`** | **The user-safe override point.** Redefine a bundled macro here. |
| **`user-hooks.cfg`** | **User callback hooks** (e.g. `_USER_PRINT_START`) invoked by the bundled macros. |

### How to change macro behavior — safely

- **Redefine a whole macro:** copy the `[gcode_macro NAME]` block into `overrides.cfg` and
  edit it there. It wins because it's included later. This survives updates; editing the
  bundled file does not.
- **Hook into print start/end/pause:** put your G-code in the matching hook in
  `user-hooks.cfg`. The bundled macros call these hooks so you don't have to fork them.
- **Small tweak (speeds, temps):** prefer variables. Many macros read `variable_*` from a
  `[gcode_macro _RATICAL_VARS]`-style holder or from `printer.cfg` — set the variable rather
  than editing logic.

### Apply changes

```bash
# on the Pi
RESTART            # or: systemctl restart klipper ; or FIRMWARE_RESTART in the console
```

No rebuild or registration is needed for macros.

### Gotchas

- **Don't edit files marked `# WARNING. DO NOT EDIT THIS FILE.`** — put changes in
  `overrides.cfg` / `user-hooks.cfg`.
- **Include order matters.** Overrides must be included *after* the file they override;
  `overrides.cfg` already is. If you add a new cfg, mind where it's `[include]`d.
- A macro that calls a custom command (e.g. `BEACON_RATICAL_CALIBRATE`) depends on the
  matching **klippy extension** (Part B) being loaded.

## Part B — klippy extensions (`configuration/klippy/*.py`)

These are Python modules that Kalico loads as `klippy/extras/<name>.py` (or
`klippy/kinematics/<name>.py`). Ratical's set:

| Module | Adds |
|---|---|
| `ratical.py` | The general-purpose `[ratical]` object: meshing helpers, Beacon region math, downloads, misc commands. Large; adapted from upstream. |
| `ratical_homing.py` | Custom `G28` (safe-z-home + homing-override combined, z-hop). |
| `ratical_dual_carriage_extras.py` | IDEX `dual_carriage` extras. |
| `kinematics/ratical_hybrid_corexy.py` | The hybrid-CoreXY / IDEX kinematics. |
| `beacon_mesh.py` | Beacon-based bed meshing. |
| `beacon_adaptive_heat_soak.py` | Soak until the Beacon reading stabilizes. |
| `beacon_true_zero_correction.py`, `beacon_user_z_offset.py` | Beacon zero/offset helpers. |
| `named_offsets.py` | Named tool/nozzle offsets. |
| `resonance_generator.py` | Resonance test sweeps. |
| `z_offset_probe.py` | Z-offset probing. |
| `servo_enable_delay.py` | Dwell before homing so step-servos finish enable init. |
| `vaoc_led.py` | VAOC camera LED (experimental; registered on demand). |
| `fastconfig.py`, `gcode_shell_command.py` | Config-speedup / shell-command helper. |

### The registration model (why you can't just drop a `.py`)

1. **Register** the module so the `ratical` CLI tracks it:
   ```bash
   ratical extensions register klipper <name> <path-to.py>
   ratical extensions register klipper -k <name> <path.py>   # for a kinematics module
   ```
2. **Symlink** every registered module into the klipper checkout:
   ```bash
   ratical extensions symlink
   ```
   This creates `klipper/klippy/extras/<name>.py` → `configuration/klippy/<name>.py`.

The installer does both automatically: **step 30** registers the core set and **step 80**
is the order-independent safety net that registers *every* bundled `.py` (and kinematics)
then symlinks. So in practice:

```bash
cd ~/ratical && ./install.sh 30 80
RESTART
```

### Add a new extension

1. Write `configuration/klippy/my_ext.py` with the standard klippy shape:
   ```python
   class MyExt:
       def __init__(self, config):
           self.printer = config.get_printer()
           # register commands / event handlers here
   def load_config(config):
       return MyExt(config)
   ```
2. Reference it from config (a `[my_ext]` section in a printer cfg or `printer.cfg`).
3. Register + symlink: `./install.sh 80` (or the two CLI commands above), then `RESTART`.
4. Use `[my_ext]` in your config.

### Kalico API compatibility — do not skip this

Kalico's core moves under these modules. Two guards exist:

- **`scripts/check-klippy-api.py`** (run by step 30) statically flags the known
  **axis-API drift**: modern Kalico passes axis **indices** (`0,1,2`) to
  `set_position(homing_axes=…)` / `clear_homing_state(…)`, not letters (`'x','y','z'`). If
  your module still uses letters, the check fails the install instead of shipping a printer
  that can't `G28`.
- **`steps/36-kalico-compat.sh`** re-applies idempotent source patches (package imports
  `from klippy import …`, `supports_dual_carriage`, `ZMesh` signature, etc.) after every
  `git pull`.

When you add or edit an extension, **run it through both** before committing:
```bash
python3 -m py_compile configuration/klippy/**/*.py
python3 scripts/check-klippy-api.py ~/klipper configuration/klippy
```

### Gotchas

- **Stale `.pyc`:** Python doesn't reliably invalidate `__pycache__` for symlinked,
  in-place-edited modules. Steps 20/36 clear `klipper/klippy/**/__pycache__`; if a change
  isn't taking effect, clear it and restart klipper.
- **`vaoc_led` is intentionally skipped** by step 80's bulk register (experimental).
- **Attribution:** `LICENSE`/`NOTICE` require keeping the upstream copyright headers
  (Kevin O'Connor, Helge Keck, Mikkel Schmidt, Beacon, etc.) — don't strip them.
